# Vercelへのデプロイワークフロー

このワークフローは、User Value Inc.のWebサイトをVercelにデプロイする手順を説明します。

## 前提条件
- GitHubリポジトリ: `tomomini0815/User-Value-Inc`
- Vercelプロジェクト: `user-value-inc`
- ブランチ: `main`

## デプロイ手順

### 1. 変更をコミット
```bash
git add .
git commit -m "コミットメッセージ"
```

### 2. GitHubにプッシュ
// turbo
```bash
git push
```

### 3. Vercelでデプロイを確認・トリガー

**重要**: このプロジェクトはGitHubリポジトリと連携されていますが、自動デプロイが必ずしも即座に開始されない場合があります。その場合は、以下の手順で手動デプロイをトリガーしてください。

#### 手動デプロイの手順:

1. **Vercelダッシュボードを開く**
   - https://vercel.com/dashboard にアクセス
   - `user-value-inc` プロジェクトを選択

2. **デプロイページに移動**
   - 左側のメニューから「Deployments」をクリック

3. **最新のコミットを確認**
   - デプロイリストに最新のコミットが表示されているか確認
   - 表示されていない場合は、次のステップに進む

4. **新しいデプロイを作成**
   - プロジェクトページの右上にある「...」（3点リーダー）メニューをクリック
   - 「Create Deployment」を選択
   - 「Commit or Branch Reference」フィールドに以下のいずれかを入力:
     - `main` (最新のmainブランチ)
     - または特定のコミットハッシュ（例: `1ad076f`）
   - 「Create Deployment」ボタンをクリック

5. **デプロイの進行状況を確認**
   - ステータスが「Building」→「Ready」に変わるまで待つ
   - 通常、数分以内に完了

6. **デプロイ完了の確認**
   - 本番URL (https://uservalue.jp) にアクセスして変更を確認

## トラブルシューティング

### 自動デプロイが開始されない場合
- GitHubリポジトリとの連携を確認:
  - Vercelプロジェクト設定 → Git → Connected Git Repository
  - `tomomini0815/User-Value-Inc` が接続されているか確認
- 手動デプロイ（上記手順4）を実行

### 古いバージョンがデプロイされた場合
- デプロイリストで正しいコミットハッシュが選択されているか確認
- 必要に応じて、特定のコミットハッシュを指定して再デプロイ

## 注意事項
- `main` ブランチへのプッシュが本番環境にデプロイされます
- デプロイ前に必ずローカルでテストを実行してください
- 重要な変更の場合は、デプロイ後に本番環境で動作確認を行ってください
